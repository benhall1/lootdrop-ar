## Vision

**One-liner:** The local commerce layer of the physical world — merchants place virtual loot boxes at their locations, consumers discover them via an interactive radar/map, claim real coupons, and drive foot traffic.

**Consumer experience:** Open the app → see a beautiful map with pulsing markers → tap one → radar zooms in showing nearby loot boxes → walk toward one → claim it → get a real coupon → show it at the business.

**Merchant experience:** Sign up → create a loot drop (location, coupon, schedule) → watch real-time analytics (views, claims, redemptions) → adjust budget → repeat.

**Revenue model:** Free for consumers. Merchants pay per-redemption ($0.50-$2) or monthly subscription for unlimited drops. Premium consumer tier removes ads, gives early access.

## Architecture

```
PWA (Expo Web)
├── Radar View (animated discovery)
├── Leaflet/Mapbox (real map)
├── Merchant Dashboard (create drops, view stats)
├── Social (leaderboards, share-a-deal)
├── Gamification (streaks, XP, badges)
└── Sound + Haptics
        │
        │ Supabase Client (REST + Realtime)
        ▼
Supabase (Auth + PostgreSQL + Realtime + Storage + Edge Functions)
        │
        ▼
Fly.io (Express API)
├── Stripe webhooks
├── Push notification dispatch (Web Push + email)
├── Merchant analytics engine
└── Social feed aggregation
```

**Key tech decisions:**
- **Web-first (PWA)** — faster to market, no App Store review, instant updates, shareable URLs
- **Supabase Auth** — Apple + Google + Email, handles sessions and roles
- **Radar View** instead of AR camera — works on all browsers, unique differentiator
- **PostGIS** spatial indexes for nearby box queries
- **Server-side distance validation** — blocks casual GPS spoofing (100m radius)

## Security

- Server-side distance validation on all claims (100m radius)
- Rate limiting on claim, auth, and merchant endpoints
- Input sanitization on all merchant-provided content
- Coupon codes: high-entropy, one-time-use, server-validated
- Supabase RLS policies for user data isolation
